==================================================
IMPLEMENTATION RULES
==================================================

1. Never make architecture-breaking changes silently.
Any meaningful change must be reflected in:
- docs/final_specs.md
- docs/tasks_breakdown.md
- system/development_log.md

2. Never implement large ambiguous chunks at once.
Break work into atomic tasks first.

3. Never let the extension and EXE drift into separate products.
All shared logic must live in shared packages/modules.

4. Heavy scraping logic must stay portable.
Do not hardwire core scraping logic only into extension code or only into desktop code.

5. Keep deployment routes separate where necessary, but preserve shared contracts.
It is acceptable to have separate adapters for:
- self-hosted object storage vs cloud object storage
- local queue vs remote queue
- local browser sidecar vs remote browser workers
But contracts must remain unified.

6. AI must not replace deterministic engineering.
Use AI for:
- routing
- repair
- normalization
- dedupe
Do not rely on it for all extraction by default.

7. Prioritize production-grade structure over speed hacks.
