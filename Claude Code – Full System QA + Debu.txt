REQUIRED TESTING METHODOLOGIES

The QA process must incorporate the following testing approaches.

1. Smoke Testing

Perform a high-level sanity check of the platform.

Verify that the critical flows work:

• login
• admin dashboard load
• client dashboard load
• middleware routing
• sidebar rendering
• API responses

Goal:

Ensure the system is fundamentally operational.

2. End-to-End (E2E) Testing

Replicate real user workflows.

Test full journeys including:

Admin user:

login → middleware → admin route → admin layout → dashboard render → API data

Client user:

login → middleware → dashboard route → client layout → dashboard render → client data

Verify the entire flow works as expected.

3. Use-Case / Scenario Testing

Test from the user’s perspective.

Admin scenarios:

• viewing dashboard
• triggering scans
• viewing products
• viewing clients
• viewing analytics

Client scenarios:

• viewing products
• submitting requests
• viewing insights

Confirm the UI and APIs behave correctly for each scenario.

4. Integration Testing

Verify interaction between components:

Frontend → API routes
API routes → Supabase
API routes → backend worker
Frontend → Supabase direct queries

Confirm data flow works correctly between modules.

5. Regression Testing

Ensure recent work has not broken existing behavior.

Pay special attention to:

• RBAC changes
• SQL changes
• middleware updates
• dashboard rendering

Confirm older working features still behave correctly.

LIVE DATA TESTING

Where possible, test flows with real runtime data.

Verify:

• login flow
• admin dashboard load
• client dashboard load
• Supabase queries
• API responses
• backend queue responses

If runtime testing is not possible:

Clearly state:

• what was attempted
• what dependency is missing
• what remains unverified

Do not simulate results.
